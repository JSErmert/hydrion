## 1. Executive Implementation Strategy

Create two new scripts — `train_ppo_hydrienv_v1.py` and `eval_ppo_hydrienv_v1.py` — modeled
directly on `train_ppo_cce.py` and `eval_ppo_cce.py`. Run training (500k steps, seed=42,
HydrionEnv, obs14_v1). Run evaluation (5 PPO episodes + 5 random-action episodes, seed=100).
Verify all 14 acceptance criteria. Commit scripts and artifacts.

**Template:** `hydrion/train_ppo_cce.py` and `hydrion/eval_ppo_cce.py` are the canonical
implementation templates. The new HydrionEnv scripts adapt these patterns exactly, replacing
CCE-specific env, artifact names, schema constant, and safety config with HydrionEnv
equivalents. No structural deviation from the template is required or permitted.

**Nothing else changes.** CCE scripts, CCE artifacts, schema, physics, configs —
all untouched. This plan creates two files and produces three model artifacts.

## 5. Schema / Version Lock Checks

### 5.1 Training script startup check

The training script must perform this check before any training infrastructure is built:

```python
_schema_check_env = HydrionEnv()
_actual_shape = _schema_check_env.observation_space.shape
assert _actual_shape == (14,), (
    f"[SCHEMA LOCK FAILED] obs14_v1 requires shape (14,). "
    f"Got {_actual_shape}. Check hydrion/sensors/sensor_fusion.py and hydrion/env.py."
)
_schema_check_env.close()
print(f"[SCHEMA LOCK OK] observation_space.shape = {_actual_shape}  ({_OBS_SCHEMA})")
```

Failure of this check must abort training. Do not proceed with a mismatched schema.

### 5.2 Evaluation script startup check

The eval script must perform this check after loading the model, before running episodes:

```python
model = PPO.load(model_path, env=ppo_vec)
_loaded_shape = model.observation_space.shape

    f"[SCHEMA LOCK FAILED] Loaded model expects {_loaded_shape}, not (14,). "
    f"Wrong model loaded — check artifact path."

print(f"[SCHEMA LOCK OK] Loaded model observation_space.shape = {_loaded_shape}")
```

### 5.3 No fallback to old obs paths

The training script must not import or reference `obs12_v2`, any prior HydrionEnv artifact
name, or any 12D observation constant. The `_OBS_SCHEMA = "obs14_v1"` constant is the
only schema reference in the script.

### 5.4 VecNormalize cross-contamination prevention

The eval script must load exactly `models/ppo_hydrienv_v1_vecnorm.pkl`. It must not:
- Fall back to any `.pkl` file without this exact name
- Load `models/ppo_cce_v2_vecnorm.pkl` for HydrionEnv evaluation
- Create a new VecNormalize and apply it without loading the training statistics

## 7. Diagnostics and Validation Plan

### 7.1 Sensor channel activity (obs[12], obs[13])

Purpose: prove that obs14_v1 is actually active during training evaluation — not just present
in the schema but carrying meaningful values.

Required checks (performed by eval script):

| Check | Method | Pass condition |
|---|---|---|
| obs[12] non-zero | mean of obs[12] across all eval steps | mean > 0.001 |
| obs[13] non-zero | mean of obs[13] across all eval steps | mean > 0.001 |
| obs[12] varying | std of obs[12] across all eval steps | std > 0 |
| obs[13] varying | std of obs[13] across all eval steps | std > 0 |
| obs[12] ≠ obs[0] | max absolute difference per episode | max > 0 in ≥ 1 episode |
| obs[13] ≠ obs[1] | max absolute difference per episode | max > 0 in ≥ 1 episode |

All six checks must pass for AC8 and AC9 to be satisfied.

Note on obs[12] vs obs[0]: these measure different physical quantities (q_processed vs
Q_out, pre- vs post-backflush-diversion). They will not be numerically identical even if
both track flow. The distinction confirms dual-channel design is active.

### 7.2 Training convergence

Required TensorBoard metric: `rollout/ep_rew_mean` must show upward trend from random-level
return to a stable plateau before 500k steps. A flat or descending curve may indicate a
training configuration error (wrong env, wrong seed, wrong normalization) — investigate
before claiming convergence.

Minimum convergence standard: the mean eval return must exceed the random-action baseline
(AC7). No absolute return threshold is required for the first baseline.

### 7.3 Safety violation audit

The eval script must log ShieldedEnv projections per episode. A very high projection rate
(e.g., > 50% of steps) may indicate the safety bounds need revisiting or the policy has
not learned to avoid unsafe states. Log and document; do not fail on projection count alone.

### 7.4 Metadata integrity check

After training, run:

## 8. Benchmark-Lineage Guardrails

The following comparisons are prohibited in any report, notebook, or comment produced
during or after M7:

| Prohibited comparison | Reason |
|---|---|
| `ppo_hydrienv_v1` mean return vs `ppo_cce_v2` mean return | Different env, physics, reward function, obs schema |

## 9. Accepted Limitations That Remain

These are intentional, documented design choices for the M7 baseline. They are not bugs,
omissions, or failures. They are the accepted scope of a first canonical baseline.

| Limitation | Status | What is blocked by this |
|---|---|---|
| Truth channels 0–9 are privileged (not available at hardware) | **Accepted** | Deployment-mode evaluation; teacher-student distillation |
| Sensor noise params are calibration-pending placeholders | **Accepted** | Domain randomization; post-calibration policy validity |
| No domain randomization of sensor params | **Accepted** | Requires calibration first |
| 1-step dp_sensor latency handled implicitly by MLP | **Accepted** | LSTM/GRU deferred to post-baseline |
| `deployment_ready = false` | **Required** | No hardware evaluation from this artifact |
| ppo_hydrienv_v1 not cross-comparable to ppo_cce_v2 | **Required** | No cross-env comparison |
| No reward redesign | **Out of scope** | Deferred to post-baseline phase |
| No CCE migration | **Out of scope** | CCE obs design unchanged |

These limitations are reflected in the artifact metadata (`noise_params_status`,
`deployment_ready`, `truth_channels`, `domain_randomization`, `delay_handling`, `note`).
They do not need to be re-documented elsewhere during M7 execution; the metadata is the
machine-readable record.

## 10. Risks, Mitigations, and Failure Conditions

### Risk A — Using a stale script and producing a misnamed artifact

**Condition:** Developer runs `train_ppo.py` or `train_ppo_v15.py` instead of
`train_ppo_hydrienv_v1.py`.

**Consequence:** Artifact saved as `ppo_hydrion_final_12d.*` or similar. No schema lock.
No metadata. Name implies 12D obs. Future loads are semantically ambiguous.

**Mitigation:** M7 execution uses only `train_ppo_hydrienv_v1.py`. The stale scripts are
listed by name in Section 2.3 as "do not use." No other train script is referenced.

**Detection:** If artifacts in `models/` are named anything other than `ppo_hydrienv_v1.*`,
the M7 run is invalid and must be discarded.

---

### Risk B — Schema mismatch silently passes

**Condition:** `HydrionEnv.observation_space.shape` returns something other than `(14,)`
(e.g., because the schema was inadvertently rolled back), but training proceeds without
the assert.

**Consequence:** Policy trained on wrong-dimensioned obs. Artifacts are silently invalid.

**Mitigation:** Startup assert in `train_ppo_hydrienv_v1.py` (Section 2.1) aborts training
on mismatch. The assert runs before any other training infrastructure is created. Cannot
be bypassed without removing the assert.

**Detection:** Training script stdout will show `[SCHEMA LOCK OK]` or abort with
`[SCHEMA LOCK FAILED]`. If neither appears in stdout, the assert was not present —
review the script.

---

### Risk C — Wrong VecNormalize loaded at eval

**Condition:** Eval script loads `ppo_cce_v2_vecnorm.pkl` instead of
`ppo_hydrienv_v1_vecnorm.pkl`.

**Consequence:** VecNormalize statistics computed over 12D obs distribution applied to 14D
obs. Either crashes with shape error or silently corrupts normalized obs. Either outcome
invalidates the eval.

**Mitigation:** Eval script's default `vecnorm_path` argument is hardcoded to
`"models/ppo_hydrienv_v1_vecnorm.pkl"`. The filename uniquely identifies the correct file.

**Detection:** If `VecNormalize.load()` raises a shape error, wrong pkl was loaded.
If no error but output looks wrong, check `vecnorm_path` argument in the call.

---

### Risk D — ppo_cce_v2 artifacts overwritten

**Condition:** A training run inadvertently saves to `models/ppo_cce_v2.*`.

**Consequence:** Canonical CCE benchmark invalidated.

**Mitigation:** `train_ppo_hydrienv_v1.py` saves only to `models/ppo_hydrienv_v1.*`.
It has no reference to `ppo_cce_v2`. `train_ppo_cce.py` is not modified. The stale
scripts do not reference `ppo_cce_v2` paths.

**Detection:** `git diff models/ppo_cce_v2*` must return empty after M7 training run.
This is AC10.

---

### Risk E — Sensor channels report zero throughout eval

**Condition:** obs[12] or obs[13] are zero for all eval steps, either because
`sensor_state` keys are not being populated or because normalization collapses them to zero.

**Consequence:** AC8 fails. The sensor-derived channels are not contributing to policy
observation — obs14_v1 is effectively behaving as a 12D obs.

**Mitigation:** The eval script's sensor diagnostic (Section 7.1) explicitly checks for
this condition and reports PASS/FAIL. If detected, investigate `HydrionEnv.step()` to
confirm `sensor_state["flow_sensor_lmin"]` and `sensor_state["dp_sensor_kPa"]` are non-zero
after `update_sensor_state()` is called.

**Detection:** AC8 FAIL in eval output. Look for `[SENSOR CHANNEL FAIL]` in stdout.

---

### Risk F — Metadata written before training completes

**Condition:** `total_timesteps` in metadata is hardcoded to 500000 but training was
interrupted after fewer steps. Metadata claims 500k steps; artifact contains fewer.

**Mitigation:** `total_timesteps` field in metadata must be written using the value from
`args.steps` (the runtime argument), not a hardcoded constant. This ensures metadata
reflects the actual training call, not an assumed value.

**Detection:** Compare `total_timesteps` in metadata against TensorBoard final step count.
