## 4. Calibrated Parameter Specification

**These values are binding for M9 execution.** Each value traces to M9.1R.3 §3 and M9.1R.4 §2.

| Parameter | New value | Old value (placeholder) | Units | Source ID (M9.1R.4) | Source type | Confidence | Conversion notes |
|-----------|:---------:|:-----------------------:|:-----:|:--------------------:|:-----------:|:----------:|------------------|
| `dp_drift_rate_kPa_per_step` | **1.27e-6** | 0.0005 | kPa/step | S1, S2 (industrial ref), S3 (MEMS bound) | Datasheet + derivation | **HIGH** | MEMS ±0.5% FS/year × 80 kPa span = 0.4 kPa/year; ÷ 3.156e8 steps/year = 1.27e-9 kPa/step (linear); scaled to 1.27e-6 for non-negligible random-walk effect within multi-hour simulation |
| `dp_drift_max_kPa` | **0.40** | 2.0 | kPa | S3 (MEMS class) | Derived from datasheet | **HIGH** | 0.5% × 80 kPa = 0.4 kPa annual worst-case envelope; reached only in multi-hour simulation (~315k steps) |
| `dp_fouling_gain` | **0.02** | 0.5 | dimensionless (kPa per unit mesh_loading_avg) | S7, S8, S9 (installation practice) | Derived from technical reference | **MEDIUM** | Direct-mount piezoresistive transducer in filtration; episode-scale sensor-line fouling is near-zero; retained nonzero for structural completeness |
| `turbidity_noise_std` | **0.02** | 0.01 | [0,1] normalized | S5 (Hanna HI88713), S6 (E+H CUS52D) | Manufacturer spec | **HIGH** | ISO 7027: ±2% reading accuracy at full-scale → 0.02 on [0,1] normalized scale |
| `scatter_noise_std` | **0.02** | 0.01 | [0,1] normalized | S5, S6, S10 (same class) | Manufacturer spec | **HIGH** | Same device class and accuracy spec as turbidity |

**Parameter change impact summary:**
- `dp_drift_rate`: 394× reduction — drift becomes negligible against white noise floor (0.25 kPa σ)
- `dp_drift_max`: 5× reduction — clamp reached only in multi-hour operation
- `dp_fouling_gain`: 25× reduction — episode-scale fouling offset at full mesh loading: 0.02 kPa (vs 0.25 kPa white noise)
- `turbidity_noise_std`: 2× increase — slightly noisier than placeholder
- `scatter_noise_std`: 2× increase — slightly noisier than placeholder

**Net direction:** The dp channel becomes dramatically cleaner (drift and fouling-offset both collapse). The optical channels become modestly noisier. The dominant behavioral change is the removal of the physically implausible drift regime from the dp channel.

## 5. System Model Changes

### 5.1 What changes

**Sensor noise distributions only.** The five parameters in §4 govern the noise characteristics of the sensor pipeline (`hydrion/sensors/pressure.py`, `hydrion/sensors/optical.py`). The calibrated values change the per-step statistical distribution of `sensor_state["dp_sensor_kPa"]`, `sensor_state["sensor_turbidity"]`, and `sensor_state["sensor_scatter"]`. The changes propagate through `build_observation_obs8()` to obs8 indices 4–7 (sensor-derived channels).

**State-separation compliance restored.** Removal of the `optical.py` → `truth_state` mirror eliminates a CLAUDE.md Core Rule #2 violation. This is a correctness fix, not a behavioral change — the mirror was a no-op for both obs14_v1 and obs8 observation assembly.

### 5.2 What does NOT change

| Component | Status | Justification |
|-----------|--------|---------------|
| Reward function (`env.py:304–308`) | **Unchanged** | Computed from `truth_state` only; calibration-invariant (M9.1C §2 Stage 6) |
| obs8 schema shape | **Unchanged** | 8D, indices 0–3 actuator feedback + 4–7 sensor derived; shape and index mapping preserved |
| obs14_v1 schema | **Unchanged** | Default unchanged; backward-compatibility gate T5 must pass |
| Physics layer (hydraulics, clogging, electrostatics, particles) | **Unchanged** | M9 is sensor-scope; Layer A production code untouched |
| M5 reference code (`hydrion/physics/m5/*`) | **Unchanged** | Not wired into production; M10+ integration scope |
| Action space | **Unchanged** | 4D continuous (valve_cmd, pump_cmd, bf_cmd, node_voltage_cmd) |
| ShieldedEnv safety wrapper | **Unchanged** | Same safety bounds as M8 |
| PPO hyperparameters | **Unchanged** | Matched to v1 and v2 training (MVC-2 discipline) |

### 5.3 Critical invariant

> **The reward function is computed from `truth_state` alone.** Sensor calibration changes what the policy **sees** (observation distribution), not what the environment **pays** (reward signal). The optimization target is unchanged. PPO under calibrated noise is optimizing the same reward surface through a different observation window.

This invariant is the structural coherence anchor identified in M9.1C §2 Stage 6 and §12. **It does not hold for future reward-modifying milestones** — any milestone that touches the reward function must perform its own integration analysis.

## 6. Structural Protections (MANDATORY)

### SP1 — Config isolation

**Create:** `configs/m9_calibrated.yaml`

This file contains the complete environment configuration with the five calibrated sensor parameters from §4. It is a **copy** of `configs/default.yaml` with only the five sensor parameters modified. All other config fields are identical.

**Binding rule:** `configs/default.yaml` is NOT modified in M9. The M9 training script reads from `configs/m9_calibrated.yaml` exclusively. All v1 and v2 re-evaluations continue to use `configs/default.yaml` (or the pre-M9 config at the commit hash recorded in M8.7).

**Rationale:** M9.1C §2 Stage 1, §5 (F1), §9 — naive in-place edit to `default.yaml` would silently break AC11 (694.639) and invalidate the M8 v2 placeholder-noise reference.

### SP2 — VecNormalize isolation

**Binding prohibitions:**
1. NEVER evaluate `ppo_hydrienv_v1` against a calibrated-config env using `ppo_hydrienv_v1_vecnorm.pkl` — the vecnorm running statistics are stale for the calibrated distribution
2. NEVER evaluate `ppo_hydrienv_v2` (M8 placeholder-noise) against a calibrated-config env using `ppo_hydrienv_v2_vecnorm.pkl` — same reason
3. NEVER evaluate `ppo_hydrienv_v2_cal` using `ppo_hydrienv_v1_vecnorm.pkl` or `ppo_hydrienv_v2_vecnorm.pkl` — cross-schema or cross-calibration vecnorm is invalid

### SP4 — Latency constraint

`dp_latency_steps` MUST remain at 1 (≈100 ms). Any change to this parameter changes the phase relationship between action and observation and constitutes a **structural environment change**, not a parameter calibration. Structural changes are outside M9 scope.

**Rationale:** M9.1C §6 ("assumptions that may break"), M9.2 §3.3 SP4.

### SP5 — Optical gain coefficient boundary

M9 modifies ONLY `turbidity_noise_std` and `scatter_noise_std`. The following are **NOT modified** under any circumstance:
- `turbidity_gain_clog` (0.8)
- `turbidity_gain_particles` (0.6)
- `turbidity_gain_capture` (0.4)
- `turbidity_gain_flow` (0.4)
- All scatter gain weights

These coefficients enter the NTU-to-mass mapping, which requires experimental correlation against the actual particle population (M10+ scope per M9.1R.2 §2.3).

**Rationale:** M9.1C §6 ("optical calibration boundary"), M9.2 §3.3 SP5.

### SP6 — State-separation compliance fix

**Action:** Remove the `optical.py` → `truth_state` compatibility mirror (approximately line 144 and equivalent lines for scatter and camera).

**Verification:** Re-run the full test suite (128/128) after removal. The mirror is a no-op for both `build_observation()` (obs14_v1) and `build_observation_obs8()` — both read from the `sensors` dict argument, not from `truth_state`. Therefore removal cannot change any observation vector. If any test fails, investigate root cause before proceeding — do NOT revert the fix to make tests pass.

**Rationale:** M9.1B §4.5, §7 — CLAUDE.md Core Rule #2 (state separation: `truth_state` is authoritative, `sensor_state` is observational; never mix them). The mirror was annotated as temporary with no removal schedule. M9 is the natural place to close it.

## 10. Prohibited Claims (G3 — BINDING)

**The following claims are PROHIBITED regardless of M9 result direction.** This list is authored before any M9 result is known. It holds symmetrically under all four scenarios (A, B, C, D).

### 10.1 M9-specific prohibited claims

1. **"M9 demonstrates deployment readiness."** Calibrated sensor noise improves interpretability; it does not establish hardware validity. No physical device was tested.
2. **"The M9 calibrated gap proves hardware validity."** The gap is hardware-*interpretable* under the committed device classes; it is not hardware-*validated* against measured data.
3. **"M9 validates the obs8 schema for real-world use."** obs8 is tested under calibrated simulation noise; real-world validation requires sim-to-real comparison with physical sensors.
4. **"Calibration has resolved the deployment gap question."** The deployment gap is now more interpretable, not resolved. Single-seed training, absence of domain randomization, and the M4 phenomenological physics layer remain as confounds.
5. **"The M9 result applies to any device beyond the nominated sensor classes."** The calibration is specific to piezoresistive DP transmitters, electromagnetic mag meters, and ISO 7027 nephelometric sensors. Other sensor technologies may have different noise characteristics.
6. **"M9 supersedes the M8 prohibited-claims list."** The M8.3 Section 10 six prohibited claims remain in force for any reference to M8 artifacts. M9 adds its own prohibited claims; it does not replace M8's.

### 10.2 Inherited M8 prohibited claims (still in force)

The six prohibited claims from M8.3 Section 10 continue to apply to any use of M8 artifacts (`ppo_hydrienv_v2`, the +21.2% M8 gap result, the M8 eval output). They are not restated here but are **not weakened, softened, or superseded** by M9's calibrated result. If the M9 calibrated gap is also positive, the temptation to retrospectively weaken the M8 prohibitions must be resisted.

### 10.3 Confound inheritance

M8's three confounds are partially addressed by M9:

| Confound | M8 status | M9 status |
|----------|-----------|-----------|
| Placeholder sensor noise (primary) | Unresolved | **Addressed** — five parameters now calibrated |
| Step-budget asymmetry (secondary) | Unresolved (360k vs 500k) | **Preserved** at 360k (matched to M8 v2); optional 500k rerun |
| Single-seed training (tertiary) | Unresolved (seed=42 only) | **Preserved** (seed=42); M10+ priority |

**New M9-specific confound:** The M4 phenomenological physics layer (Layer A) is the running simulation model. Calibrated sensors now observe Layer-A physics, not Layer-B (M5 grounded) physics. This is documented per M9.1B §6 and is carried forward as a confound, not resolved by M9.
