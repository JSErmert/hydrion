## 6. Key Findings

### 6.1 The M8 deployment gap survives calibration

The +21.2% M8 gap under placeholder noise was not a noise-regime artifact. Under calibrated noise:
- Config A: +22.0% (slightly larger than M8)
- Config B: +18.2% (slightly smaller than M8)
- Range: **+18 to +22%** — the M8 result falls within this band

This means the obs8 policy's advantage over obs14 is a genuine information-content or dimensional-reduction effect, not a consequence of placeholder noise characteristics.

### 6.2 The M8 hyperparameters were noise-regime-dependent

The original PPO hyperparameters (lr=3e-4, clip=0.2) converged under placeholder noise but failed under calibrated noise. This is a genuine sensitivity finding:
- Placeholder dp drift (0.0005 kPa/step) provided temporal structure that helped PPO's value function
- Calibrated dp drift (1.27e-6 kPa/step) removed that structure, making the optimization landscape harder
- The fix was straightforward: lower learning rate and tighter clipping

**For future training under calibrated noise:** Use lr ≤ 1e-4 and clip_range ≤ 0.1. The M8 hyperparameters should not be assumed transferable across noise regimes.

### 6.3 Config A is the recommended canonical artifact

Config A (lr=1e-4, clip=0.1, batch=128, epochs=15, 500k steps) outperforms Config B (lr=3e-5, clip=0.1, batch=256, epochs=20, 1M steps) at less than half the training time. It produces the highest absolute return (847.7) and the largest calibrated gap (+22.0%).

**Recommended M9 canonical artifact: `ppo_hydrienv_v2_cal_A`**

### 6.4 The initial M9.5 Scenario D was a hyperparameter issue, not an architecture issue

PPO + MLP is sufficient under calibrated noise. No recurrent architecture (LSTM/GRU) or architectural change is needed. The failure was a hyperparameter mismatch that was resolved by a controlled correction within the same architecture.
