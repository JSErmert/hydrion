## 5. Core Integration Logic

### 5.1 Toggle structure

**File:** `hydrion/physics/particles.py`

**Location:** `ParticleModel.__init__()` (after line 223)

```python
# M10: capture mode toggle
physics_raw = {}
if cfg is not None and hasattr(cfg, "raw"):
    physics_raw = getattr(cfg, "raw", {}).get("physics", {}) or {}
self._capture_mode = str(physics_raw.get("capture_mode", "m4"))
```

**Location:** `ParticleModel.update()` (replacing lines 286-291)

```python
if self._capture_mode == "m5_rt":
    # M10: RT single-collector capture per stage
    from .m5.capture_rt import rt_single_collector, stage_capture_efficiency
    from .m5.capture_rt import MESH_S1, MESH_S2, MESH_S3_MEMBRANE

    d_p_m = p.eta_ref_d_um * 1e-6  # µm → m conversion
    Q_m3s = max(Q_lmin / 60000.0, p.eps)
    stage_vol_L = float(state.get("stage_volume_L", 0.25))
    U_ms = Q_m3s / (stage_vol_L * 1e-3)  # face velocity [m/s]

    rt_s1 = rt_single_collector(d_p_m, MESH_S1, U_ms, p.rho_dense_kgm3,
                                 p.rho_water_kgm3, p.mu_water_Pas, H=0.5e-20)
    eta_s1 = stage_capture_efficiency(rt_s1["eta_0"], MESH_S1, MESH_S1.d_c_m)

    rt_s2 = rt_single_collector(d_p_m, MESH_S2, U_ms, p.rho_dense_kgm3,
                                 p.rho_water_kgm3, p.mu_water_Pas, H=0.5e-20)
    eta_s2 = stage_capture_efficiency(rt_s2["eta_0"], MESH_S2, MESH_S2.d_c_m)

    rt_s3 = rt_single_collector(d_p_m, MESH_S3_MEMBRANE, U_ms, p.rho_dense_kgm3,
                                 p.rho_water_kgm3, p.mu_water_Pas, H=0.5e-20)
    eta_s3 = stage_capture_efficiency(rt_s3["eta_0"], MESH_S3_MEMBRANE,
                                       MESH_S3_MEMBRANE.d_c_m)

    # MANDATORY GUARD: RT N_G embeds gravity; Stokes boost would double-count
    capture_boost_settling = 0.0
else:
    # Existing M4 power-law path (unchanged)
    eta_s1 = _capture_eff_s1(d_um, ff_s1, p.fouling_gain_s1)
    eta_s2 = _capture_eff_s2(d_um, ff_s2, p.fouling_gain_s2)
    eta_s3 = _capture_eff_s3(
        d_um, ff_s3, p.fouling_gain_s3,
        Q_lmin, p.s3_flow_penalty_coeff, p.s3_flow_onset_lmin,
    )
    # M4 Stokes settling (existing path, unchanged)
    d_m = d_um * 1e-6
    v_s = stokes_velocity_ms(p.rho_dense_kgm3, d_m, p.rho_water_kgm3, p.mu_water_Pas)
    Q_proc_Ls = max(Q_lmin / 60.0, p.eps)
    stage_vol_L = float(state.get("stage_volume_L", 0.25))
    t_res_s = stage_vol_L / Q_proc_Ls
    capture_boost_settling = float(np.clip(
        v_s * t_res_s / max(p.stage_height_m, p.eps), 0.0, 0.05
    ))
```

### 5.2 Design decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| RT density | PET only (`rho_dense_kgm3 = 1380.0`) | Phase 1 minimal scope; matches M4 dense-phase-only approach |
| Bed depth | `d_c_m` per stage (single-screen) | Avoids deep-bed saturation; physically correct for single mesh layer |
| Hamaker constant | `H = 0.5e-20 J` | From `m5/materials.py` — conservative lower bound (Visser 1972, Gregory 1981) |
| Import location | Deferred inside `"m5_rt"` branch | Prevents M5 side-effects under `"m4"` mode; isolation proof |
| Face velocity | `U = Q / (stage_volume × 1e-3)` | Approximation using stage volume as cross-section proxy; matches M4 residence time computation |
| Fouling coupling | None under `"m5_rt"` | RT is a clean-filter formula; fouling coupling deferred to Phase 2 |

### AC1: M4 isolation

Running with `configs/m9_calibrated.yaml` after M10 code changes produces bit-identical results to pre-M10 behavior at fixed seed.

**Verification:** Step-by-step comparison of `C_out`, `particle_capture_eff`, `eta_system`, reward trajectory at seed=42. Zero divergence at machine precision.

### AC2: AC11 gate

`ppo_hydrienv_v1` evaluated under `configs/default.yaml` produces AC11 = 694.639 at all five checkpoints: pre-exec, post-config, pre-train, post-eval, closeout.

### AC3: Stokes guard

`capture_boost_settling == 0.0` at every step when `capture_mode == "m5_rt"`.

**Verification:** Assert in test. Nonzero = FAILURE.

### AC4: RT activation

Under `capture_mode == "m5_rt"`:
- `capture_eff_s1/s2/s3` differ from M4 values
- `C_out` differs from M4 values
- `particle_capture_eff` differs from M4 values
- obs8 indices 4-5 differ from M4 values

Identity with M4 = FAILURE (RT not active).

### AC5: Reward invariance

Reward trajectory matches between `"m4"` and `"m5_rt"` at fixed seed within floating-point tolerance.

**Verification:** Per-step reward comparison at seed=42 under identical action sequences. Any divergence = FAILURE.

### AC6: Clogging invariance

`mesh_loading_avg` and `fouling_frac_s1/s2/s3` trajectories match between `"m4"` and `"m5_rt"` at fixed seed.

**Verification:** Per-step comparison. Any divergence = FAILURE. (Confirms C_fibers=1.0 severs coupling.)

### AC7: Channel-level obs8 verification

| obs8 index | Expectation |
|:----------:|-------------|
| 0-3 | MUST match between M4 and M5 |
| 4-5 | MUST differ between M4 and M5 |
| 6-7 | MUST match between M4 and M5 |

### AC8: RT plausibility

RT diagnostic outputs at reference conditions (d_p=10 µm, PET, Q=13.5 L/min):
- `eta_0 ∈ [0, 1]`
- N_G > 0 for PET (ρ=1380 > ρ_water=1000)
- Diffusion term dominates at small d_p; interception dominates at large d_p

### AC9: Policy training convergence

`ppo_hydrienv_v2_m10_rt` trains to comparable return as M9 canonical (within M9 Config A/B range: 694-848). No higher return required. Non-convergence triggers investigation, not automatic failure.
