# M10 Output Contract

**Governed by:** AROD Schema v1 §16 (chat echo requirement)
**Scope:** M10 and all future milestones unless superseded

---

## 1. When artifacts are printed

Immediately after writing each artifact to disk — not on request.

The AROD pattern is: save → print → next artifact. Every markdown artifact is printed in chat the moment it is created. The user should never need to ask for output.

---

## 2. What is printed per prompt execution

Each milestone prompt execution produces artifacts in this order:

1. **Prompt file** (saved only — the user already has the prompt content)
2. **Primary output** (saved AND printed)
3. **Insight file(s)** (saved AND printed)

After all artifacts are printed, return the structured return block:
- prompt path
- primary output path
- insight path(s)
- short summary (one to three sentences)
- next artifact to expect

---

## 3. How artifacts are printed

Each artifact is printed as a single, copy-pasteable fenced block using a **4-backtick fence** (` ```` `). This prevents nesting collisions — inner triple-backtick blocks (python examples, ASCII diagrams) do not terminate the outer 4-backtick fence.

Format per artifact:

1. Level-2 heading: `## <file path>`
2. Open 4-backtick fence: ` ````markdown `
3. Full file contents — unmodified, no truncation, no summary
4. Close 4-backtick fence: ` ```` `

This satisfies AROD Schema v1 §16 (copy-pasteable backup, immediate inspection) without the rendering problem of plain-text output or the collision problem of triple-backtick fencing.

If multiple artifacts are printed in sequence, separate them with a horizontal rule (`---`).

---

## 4. What is NOT printed

- Prompt files (the user provided the prompt — echoing it back wastes context)
- Files from prior milestones unless explicitly requested
- Summaries, truncations, or paraphrases in place of full content

---

## 5. Registry

There is no static artifact registry. The AROD phase sequence (§6 of the schema) defines what artifacts exist:

```
.0  prompt           (saved, not printed)
.1  primary output   (saved + printed)
    insight(s)       (saved + printed)
```

Additional phases (.1A corrections, .1R research, .3 execution docs, etc.) follow the same rule: save then print the output, skip printing the prompt.

---

## 6. Conflict resolution

If printing the full artifact would exceed a practical output limit:
- Print the primary output in full (highest priority)
- Print insights in full (second priority)
- If both together are too large, print the primary output only and list insight paths

The primary output is never truncated or summarized.
