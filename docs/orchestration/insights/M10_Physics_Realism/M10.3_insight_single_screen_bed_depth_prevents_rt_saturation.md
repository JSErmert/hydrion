# M10.3 Insight — Single-screen bed depth prevents RT saturation at Stage 3

**Source:** M10.3 execution document — design decision for bed_length_m parameter
**Date:** 2026-04-16
**Milestone:** M10_Physics_Realism
**Iteration:** M10.3
**Insight class:** physics integration decision

---

## The insight

> The RT deep-bed formula `E = 1 - exp(-4 alpha eta_0 L / (pi d_c))` saturates to ~1.0 for any bed length L >> d_c. At Stage 3 (`MESH_S3_MEMBRANE`, d_c = 1.5 µm), using the cone slant length (~0.04 m) as bed depth gives L/d_c ≈ 27,000 — the exponential term vanishes and every particle is captured regardless of physics. This eliminates the entire purpose of integrating RT: the formula returns 1.0 for all conditions, making M4 and M5 indistinguishable at Stage 3.

**Fix applied in M10.3:** `bed_depth_m = mesh.d_c_m` (single-screen model). This gives L/d_c = 1, producing physically meaningful capture fractions that depend on particle size, velocity, and density — which is the point of the RT integration.

**Why this matters:** Stage 3 is the dominant capture stage (5 µm membrane, highest fouling sensitivity). If RT saturates there, M10 adds grounded physics only at Stages 1-2 (where capture is already low for 10 µm particles on 500/100 µm meshes). The single-screen choice preserves RT's discriminating power at the stage that matters most.

**Caveat:** The single-screen model is an approximation. Real membrane filters may have multiple effective collector contacts per particle transit. The `conical_stage.py` module (Phase 3) handles this via configurable `bed_depth_m`. For Phase 1, single-screen is the conservative, physically interpretable choice.
