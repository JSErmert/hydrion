## 2. Implemented Changes

### Schema now active for HydrionEnv: `obs14_v1`

`build_observation()` in `hydrion/sensors/sensor_fusion.py` now returns a 14-element
`np.float32` array. `HydrionEnv.observation_space` is `spaces.Box(low=0.0, high=1.0, shape=(14,))`.

### Observation index mapping — obs14_v1

| Index | Key | Source | Physical quantity | Normalization |
|---|---|---|---|---|
| 0 | flow | truth_state | Q_out_Lmin (post-backflush-diversion output) | / 20.0 |
| 1 | pressure | truth_state | P_in (inlet absolute; encodes bypass signal) | / 80000 |
| 2 | clog | truth_state | mesh_loading_avg | — |
| 3 | E_field_norm | truth_state | radial E-field magnitude | — |
| 4 | C_out | truth_state | outlet particle concentration | — |
| 5 | particle_capture_eff | truth_state | capture efficiency | — |
| 6 | valve_cmd | truth_state | valve actuator command | — |
| 7 | pump_cmd | truth_state | pump actuator command | — |
| 8 | bf_cmd | truth_state | backflush actuator command | — |
| 9 | node_voltage_cmd | truth_state | node voltage command | — |
| 10 | sensor_turbidity | sensor_state | optical turbidity (noisy) | — |
| 11 | sensor_scatter | sensor_state | optical scatter (noisy) | — |
| **12** | **flow_sensor_norm** | **sensor_state** | **q_processed_lmin (noisy, pre-backflush-diversion)** | **/ 20.0, clip [0,1]** |
| **13** | **dp_sensor_norm** | **sensor_state** | **dp_total_pa in kPa (noisy, pure filter differential)** | **/ 80.0, clip [0,1]** |

**Indices 0–11 are unchanged from obs12_v2.** Their sources, physical quantities, and
normalization references are identical to the prior schema.

### Normalization is inline — no new sensor_state keys

Indices 12 and 13 are normalized exclusively inside `build_observation()` using `np.clip`:

```python
float(np.clip(sensors.get("flow_sensor_lmin", 0.0) / 20.0, 0.0, 1.0)),
float(np.clip(sensors.get("dp_sensor_kPa",   0.0) / 80.0, 0.0, 1.0)),
```

`sensor_state` continues to hold raw physical-unit values only (`flow_sensor_lmin` in L/min,
`dp_sensor_kPa` in kPa). No normalized keys were added to `sensor_state`. The normalization
boundary is at `build_observation()` only, preserving the sensor model / RL observation
contract separation.

## 4. Architecture Preservation

### truth_state / sensor_state separation

Intact. `build_observation()` draws indices 0–9 from `truth_state` and indices 10–13 from
`sensor_state`. No cross-contamination. No new truth_state keys were added or modified.
No new sensor_state keys were added. Normalization occurs only in `build_observation()`.

### Semantic integrity of indices 0–11

Confirmed unchanged. Index 0 still maps to `truth["flow"]` (Q_out_Lmin / 20.0). Index 1 still
maps to `truth["pressure"]` (P_in / 80000). All other indices identical to obs12_v2.

Semantic distinction preserved and documented in the `sensor_fusion.py` SEMANTIC WARNING:
- Index 0 (Q_out, post-diversion) ≠ Index 12 (q_processed, pre-diversion, noisy)
- Index 1 (P_in, inlet absolute) ≠ Index 13 (dp_total, filter differential, noisy)

### CCE

`conical_cascade_env.py` has zero diff in this phase. CCE `_obs()` is self-contained and
does not call `build_observation()`. The obs14_v1 extension has no effect on CCE.

### ppo_cce_v2

ppo_cce_v2 is trained on CCE with a 12D truth-derived observation. CCE is unchanged.
ppo_cce_v2 produces identical results on the post-M6.2B codebase as on the pre-M6.2B
codebase. CCE training infrastructure smoke test passes.

## 5. Artifact Lineage

### ppo_cce_v2

**Status: canonical, unaffected, fully valid.**

ppo_cce_v2 remains the authoritative benchmark for CCE / truth-derived physics performance.
Its input schema (12D CCE truth-derived) is unchanged. All performance records associated
with ppo_cce_v2 remain valid and comparable to any future CCE run on the unchanged schema.

### HydrionEnv obs12_v2 artifacts

Any HydrionEnv policy artifact trained under obs12_v2 (12D input) is **architecturally
incompatible** with the obs14_v1 environment (14D observation). The incompatibility is
hard — a 12D policy network cannot ingest a 14D observation vector without modification.

Any cross-schema evaluation (obs12_v2 policy on obs14_v1 env with padded/zeroed dims) is
diagnostic only and must be labeled as such. It is not a valid benchmark comparison.

Note: No canonical HydrionEnv obs12_v2 policy artifact was produced before M6.2B. The
incompatibility is stated here for structural clarity, not because an existing artifact
is at risk.

### Future HydrionEnv baseline

**Does not yet exist.**

The first canonical HydrionEnv RL baseline must be trained from scratch under obs14_v1
during Phase 3 (M7). Until that artifact exists:
- No HydrionEnv policy benchmark comparison against obs14_v1 is valid.
- ppo_cce_v2 (CCE) and the future HydrionEnv obs14_v1 artifact are independent benchmarks
  for independent environments. They are not cross-comparable.

## 6. What M6.2B Proved

1. **Add-only schema extension is the correct path for introducing sensor-derived channels.**
   Two new indices were appended without touching any existing index source, physical
   quantity, or normalization. The semantic integrity of obs12_v2 indices 0–11 was preserved
   exactly. No substitution was required or performed.

2. **Sensor-derived channels can be introduced without corrupting prior observation semantics.**
   The dual-channel design (truth-derived indices 0,1 alongside sensor-derived indices 12,13)
   is implementable and testable. Source key traceability and truth/sensor contamination
   boundaries hold under the full test suite.

3. **CCE benchmark protection was preserved without special handling.**
   Because obs14_v1 is HydrionEnv-specific and CCE's `_obs()` is self-contained, no CCE
   guard code was required beyond the existing architecture. The separation held naturally.

4. **The M6.1 sensor infrastructure (DifferentialPressureSensor, FlowRateSensor) integrates
   into the RL observation pipeline without additional architecture changes.**
   `sensor_state["flow_sensor_lmin"]` and `sensor_state["dp_sensor_kPa"]` were already
   populated from M6.1; the only new code was the two normalized read lines in
   `build_observation()`.
