# CLAUDE MACHINE VIEW V1 PROMPT

You are implementing **HydrOS Machine View v1** for the HydrOS / Hydrion system.

This is NOT a generic dashboard task.

You are building the first real visual machine interface for a **research-grade digital twin** of a **handheld laundry-outflow microplastic extraction system**.

---

## PRIMARY GOAL

Build a **machine-centered simulation interface** that is:

- clean
- grounded
- physically interpretable
- state-driven
- visually stable

---

## CORE DOCTRINE

HydrOS UI must:

- center the machine
- use real telemetry labels
- avoid placeholder values
- remain visually disciplined
- reflect physics, not decoration

---

## REQUIRED STRUCTURE

### 1. Header Bar
- HydrOS identity
- mode label
- frame/state label
- system status

### 2. Core Machine View
- inlet
- Stage 1 / 2 / 3
- outlet
- bypass path
- backflush path

### 3. Top Telemetry Band
- FLOW RATE
- PRESSURE
- BYPASS

### 4. Bottom Narrative Panel
- stage readouts
- fiber load
- pulse status
- next actions

### 5. Right Advisory Panel
- system status
- efficiency
- insights

---

## STORYBOARD STATES

Implement 5 fixed states:

1. CLEAN START
2. EARLY CAPTURE
3. RISING LOAD
4. THRESHOLD STATE
5. BACKFLUSH + AFTERMATH

These must:
- share identical layout
- differ only in state
