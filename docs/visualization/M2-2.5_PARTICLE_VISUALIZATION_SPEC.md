# HydrOS Particle Visualization Specification

## Purpose

Define how particles:
- move through the system
- interact with each stage
- accumulate
- get removed

This is NOT decorative particle effects.

This is:
> a visualization of physical transport and capture

---

# Core Principle

Particles must obey:

1. flow direction  
2. stage interaction rules  
3. conservation (no disappearing)  
4. accumulation logic  

If particles do not reflect physics, they must not be shown.

---

# Particle Types (M2)

HydrOS uses simplified classes:

## 1. Fibers (dominant)
- long, thread-like
- high clogging impact
- high capture probability

## 2. Fragments (secondary)
- small, round or irregular
- lower clogging impact
- lower capture probability

---

# Particle Flow Behavior

## Movement

Particles must:
- follow flow direction (left → right)
- match flow velocity
- accelerate/decelerate with flow changes

## Visual Encoding

- speed = flow velocity  
- density = particle concentration  
- brightness = activity / throughput  

---

# Stage Interaction

Each stage behaves differently.

---

## Stage 1 — Coarse

### Behavior
- captures large fibers
- allows most particles through

### Visual
- fibers snag and slow
- some get trapped visibly
- loose entanglement forms

### Capture rate
LOW–MODERATE

---

## Stage 2 — Medium

### Behavior
- captures medium fibers
- partially captures fragments

### Visual
- more particles slow down
- more particles stick
- buildup looks denser

### Capture rate
MODERATE

---

## Stage 3 — Fine Pleated

### Behavior
- captures majority of particles
- forms dense surface layer

### Visual
- particles slow dramatically
- many stop and accumulate
- dense “wall” or cake forms

### Capture rate
HIGH

---

# Particle Fate

Each particle must do one of:

1. pass through → exit  
2. get captured → remain in stage  
3. get removed → via backflush  

No other outcomes.

---

# Accumulation (Storage Behavior)

Captured particles must:

- remain visible in stage
- increase density over time
- reflect loading state

Optional storage model:

## Stage-local storage (M2)
- particles stay where captured

## Central reservoir (M3+)
- particles move to storage chamber

---

# Backflush Interaction

## Behavior

When backflush activates:

- some captured particles detach
- particles move in reverse direction
- particles either:
  - exit via waste
  - redistribute upstream

## Visual

- reverse particle motion
- clearing at surface
- partial removal only

---

# Conservation Rule

Total particles:


inflow = outflow + captured + removed


Must always hold visually.

---

# Density Representation

Particle density should map to:

- flow input rate  
- stage capture rate  
- fouling state  

Higher density:
- more particles per volume
- stronger visual clutter

---

# Visual Scaling Rules

Particles must not:
- overwhelm the scene
- obscure machine structure
- appear chaotic

Use:
- sampling (not every particle shown)
- density gradients
- depth fading

---

# Interaction with Fouling

Particles should:

- contribute to fouling visually
- stop moving when trapped
- become part of accumulation layer

---

# Timing Behavior

## Continuous
- particle flow is always active

## Slow accumulation
- particles settle gradually

## Event response
- backflush causes sudden movement

---

# Animation Priority

1. particle motion follows flow  
2. particle capture follows stage rules  
3. particle accumulation follows time  
4. particle removal follows backflush  

---

# M2 Implementation Scope

Include:

- flow-following particles  
- stage-specific capture  
- accumulation visualization  
- basic backflush removal  

Exclude (for now):

- full PSD modeling  
- electrostatic interaction  
- chemical effects  

---

# Future Expansion (M3)

- size-based behavior  
- fiber orientation physics  
- electrostatic attraction  
- multi-class particles  

---

# Final Definition

Particle visualization is:

> the visible explanation of system causality

Without it:
- system shows state

With it:
- system shows why the state exists
