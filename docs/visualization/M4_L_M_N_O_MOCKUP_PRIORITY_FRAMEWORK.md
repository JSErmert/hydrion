# M4 Additions (New Constraints)

M4 introduces five non-negotiable visual requirements:

## 1. η_nominal is a display contract
Must be shown as:

```
η_ref   85.4%  @ 10µm / 13.5 L/min
η_live  71.2%
Δη     -14.2 pts   cause: FLOW
```

Rules:
- qualification ALWAYS visible
- not hidden behind hover/tooltips
- not reduced to a tile
- must survive screenshot context

---

## 2. Stage capture gradient must be visible

Typical distribution:

| Stage | Capture |
|-------|---------|
| S1    | ~0–1%   |
| S2    | ~5–10%  |
| S3    | ~80%+   |

Implication:
- Stage 3 is dominant
- Stage 1 is nearly passive

Visual rule:
> Stages are NOT equal — hierarchy must be visible

---

## 3. S3 flow-rate coupling must be legible

At higher flow:
- S3 capture drops (e.g., 84% → ~65%)

Visual rule:
- flow must visibly influence Stage 3 behavior
- not static efficiency

---

## 4. Density split must be communicated

- ~15% buoyant pass-through is expected
- NOT a failure

Visual rule:
- pass-through shown in neutral/cool tone
- never red
- labeled as expected behavior

---

## 5. E-field must reflect radial physics

NOT:
- glow bands
- ambient highlights

MUST:
- reflect inner rod → outer wall geometry
- align with radial field model
