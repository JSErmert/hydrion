# First Bench Experiment SOP

## Title
Pressure–Flow–Fouling–Backflush Bench Characterization for HydrOS

## Purpose
This is the first and highest-value bench experiment for HydrOS.

Its purpose is to generate device-relevant calibration data for:

- `HydraulicsModel`
- `CloggingModel`
- `BackflushModel`
- secondarily `SensorModel`

This experiment is the shortest path from:
- proxy-supported realism
to
- HydrOS-specific external credibility

---

# 1. Primary Objectives

## Objective A — Pressure vs flow characterization
Measure how system pressure changes with flow under:
- clean state
- partially fouled state
- heavily fouled state

## Objective B — Fouling progression characterization
Measure how pressure and throughput evolve as fouling accumulates under a standardized fiber-rich surrogate load.

## Objective C — Backflush recovery characterization
Measure how much recovery is achieved per pulse / burst and how recovery changes over repeated bursts.

## Objective D — Bypass characterization
If bypass is present in the prototype, observe when it activates and how flow splits under overload.

---

# 2. Why this is the first experiment

This experiment calibrates the realism backbone first:

1. hydraulics
2. fouling
3. backflush

Without this, later realism layers (electrostatics, particle selectivity, sensor sophistication) are built on weak physical truth.

This experiment also produces the most visually and operationally valuable telemetry for HydrOS playback and review.

---

# 3. Experimental Setup

## 3.1 Required apparatus
Minimum suggested setup:

- controllable pump or flow source
- test loop with HydrOS-like staged filter assembly
- inlet pressure sensor
- outlet pressure sensor
- optional stage-to-stage pressure taps
- inline flow sensor
- reservoir / recirculation path
- controllable backflush actuation path
- data logging system
- standardized surrogate contaminant feed

---

## 3.2 Test article
The test article should approximate the current HydrOS architecture as closely as practical:

- Stage 1 coarse mesh
- Stage 2 medium mesh
- Stage 3 fine pleated stage
- nominal bypass path if relevant
- backflush pulse path

If exact geometry is not available yet, a simplified prototype is acceptable as long as geometry and deviations are documented.

---

# 4. Surrogate Load

## 4.1 Recommended first-pass surrogate
Use a repeatable fiber-heavy surrogate representing laundry outflow fouling tendency.

Initial priority is not exact chemical realism.
Initial priority is:
- repeatability
- fouling generation
- measurable pressure response

The surrogate should be documented with:
- material type
- approximate fiber size / morphology
- concentration
- mixing method

---

## 4.2 Important note
If a mixed surrogate is used later (fibers + fragments + fines), record this clearly.
The first experiment should remain as simple and controlled as possible.

---

# 5. Measurements to Record

At minimum record:

- time
- inlet pressure
- outlet pressure
- total differential pressure
- stage differential pressures if available
- processed flow
- bypass flow if measurable
- backflush command / pulse timing
- backflush cooldown timing
- total run duration
- fouling state label
- surrogate feed rate / concentration

Optional but valuable:
- optical / turbidity proxy
- high-speed or visual evidence of flow behavior
- captured mass / removed mass

---

# 6. Experimental Conditions

## 6.1 Clean-state sweep
Run flow sweeps across:
- low regime
- nominal regime
- peak transient regime

Record:
- pressure vs flow
- any instability
- onset of bypass if present

---

## 6.2 Fouling progression run
At nominal flow, introduce standardized surrogate load and run long enough to observe:
- pressure rise
- throughput degradation
- maintenance threshold crossing behavior

---

## 6.3 Backflush recovery run
Once the system reaches a fouled state:
- execute the HydrOS burst doctrine
  - 3 pulses
  - 0.4 s each
  - 0.25 s spacing
  - 8–10 s cooldown
- record recovery after each burst
- repeat for multiple bursts to measure diminishing returns

---

## 6.4 Overload / bypass run
If bypass hardware exists:
- push toward overload condition
- observe pressure threshold and flow split
- record activation and deactivation behavior

---

# 7. Output Metrics

## Hydraulics metrics
- pressure vs flow curve
- clean-state resistance estimate
- fouled-state resistance estimate
- stage-wise ΔP distribution if available
- bypass activation threshold

## Fouling metrics
- pressure-rise rate under load
- throughput decline under load
- apparent maintenance threshold onset
- stage dominance if resolvable

## Backflush metrics
- Δpressure recovery per burst
- Δflow recovery per burst
- diminishing returns across repeated bursts
- cooldown behavior
- recovery asymptote / nonzero residual fouling

## Sensor metrics
- sensor truth-vs-measurement comparison if reference instruments exist
- pressure sensor noise / lag
- flow sensor noise / lag

# 10. Success Criteria

This experiment is successful if it yields enough data to estimate, at minimum:

- clean-state pressure vs flow curve
- fouled-state pressure vs flow shift
- a meaningful maintenance threshold region
- partial, non-perfect backflush recovery
- some evidence of diminishing returns

If those are obtained, HydrOS can meaningfully improve M1.5 calibration.

---

# 11. Common Failure Modes

- surrogate load too inconsistent
- geometry too different from intended architecture
- insufficient logging resolution
- no stage pressure taps, limiting decomposition insight
- backflush hardware unable to reproduce intended pulse morphology

All deviations must be documented rather than ignored.
