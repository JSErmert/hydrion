# HydrOS Calibration Parameter Register

**[M1.5 UPDATE — 2026-04-10]** All "TBD initial" placeholders resolved. New parameters added: `dep_exponent`, `dep_floor`, `k_m1/2/3_clog`. Voltage bounds corrected. See Section 10 for change log.

This document is the authoritative register of all calibration-relevant parameters in HydrOS.

Its purpose is to make every realism-critical parameter:

- traceable
- source-backed
- milestone-scoped
- validation-aware
- bench-replaceable

This document is the bridge between:

- literature-derived initialization
- proxy assumptions
- bench experiment results
- YAML configuration updates

---

# 1. Calibration Doctrine

HydrOS calibration must respect the system roadmap:

1. Hydraulics
2. Clogging
3. Backflush
4. Electrostatics
5. Particles
6. Sensors
7. RL / reward refinement

No lower-priority subsystem should be treated as “high realism” before upstream backbone realism is stabilized.

---

# 2. Parameter Evidence Classes

Every parameter must be labeled as one of:

## Direct
Supported by directly relevant data for the same or very similar device / operating context.

## Proxy
Supported by adjacent-domain literature or engineering analogs.

## Placeholder
Currently a design assumption awaiting bench calibration.

---

# 3. Confidence Classes

## High
Strong direct evidence or highly transferable component data.

## Medium
Good proxy evidence with reasonable transferability.

## Low
Weakly transferable evidence or first-pass engineering assumption.

---

# 4. Replacement Policy

Every parameter should state when it is expected to be replaced or refined:

- stays as literature-backed default
- replace after bench hydraulics test
- replace after backflush pulse test
- replace after electrostatic ON/OFF test
- replace after sensor truth-vs-measurement test

# 10. M1.5 Change Log (2026-04-10)

The following changes were applied in the M1.5 sprint and are now reflected in this register:

## R1 — Bistable kinetics fix
- `dep_exponent`: 2.0 → **1.0**
- `dep_floor`: **0.05** (new parameter)
- Reason: dep_exponent=2 created unstable fixed points at S1≈0.667, S2≈0.417, S3≈0.222. Clean-start RL episodes would never observe fouling. dep_exponent=1 restores monotone deposition growth.

## C2 — Component sum normalization fix
- cake+bridge+pore now normalized to sum ≤ 1.0 in `_update_stage()` when overflow detected.
- No YAML parameter change; code fix only.

## R3 — Area normalization removal (pressure drop ordering)
- `k_m1_clog`: removed area factor → **2.0e7** Pa·s/m³ per unit fouling
- `k_m2_clog`: removed area factor → **8.0e7** Pa·s/m³ per unit fouling
- `k_m3_clog`: removed area factor → **3.0e8** Pa·s/m³ per unit fouling
- Reason: old code multiplied k_m*_clog by (A_s3/A_si), amplifying S1 by 7.5× and S2 by 4.1×. This made S2 the dominant ΔP source above ~17% fouling — physically wrong. Corrected: k values now directly encode pore-size physics (fine mesh >> medium >> coarse). Validated: dp_s3 >> dp_s2 >> dp_s1 at 30%/60% fouling.

## Voltage bounds fix
- `V_max_realism`: **2500 V** (was V_max=3000, violating locked 2.5 kV constraint)
- `V_hard_clamp`: **3000 V** (new separate supervisory clamp)
- Reason: locked constraint in 06_LOCKED_SYSTEM_CONSTRAINTS.md §D specifies 2.5 kV realism bound. Original code used 3000 V as operational ceiling.

## Scope lock — Option A
- Dense-phase-only scope locked in 06_LOCKED_SYSTEM_CONSTRAINTS.md §E
- Target: ρ > 1.0 g/cm³ (PET, PA, PVC, biofilm-coated)
- Excluded: PP (ρ≈0.91), PE (ρ≈0.95) — buoyant, out of scope
- Dual-path collection not planned
